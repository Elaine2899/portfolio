- 專題題目：FunCtrl - AI 驅動智慧學習地圖平台
- 畢業專題 | InnoServe 第一名 | FunCtrl 高精度影片搜尋與學習平台
- 系統目標與動機
    - 核心目標：解決 YouTube 等平台影片資訊碎片化、難以精準定位知識點的問題
    - 解決策略：透過 Semantic Vector Search 與 LLM (Gemini) 技術，實現精準的影片片段定位與動態學習地圖規劃
- 技術架構與系統驗證(我的核心貢獻)
    - 系統架構設計與選型 (Architecture & Tech Selection)
        - 技術決策：主導系統架構規劃，選擇適合的框架與 Vector Database (ChromaDB) 以支援大規模語意檢索
        - 開發流程領導：引導團隊開會流程並負責簡報製作，確保專案進度與技術目標一致
        - 基礎設施管理：負責雲端前後端部署，並在專案後期考量成本效益，將服務遷移至地端實驗室主機，展現資源管理與成本控制意識
    - 檢索演算法優化與正規化 (Algorithm Optimization)
        - 效能調優：提出以歐幾里得距離 (L2 Distance) 取代餘弦相似度，並嚴謹執行向量長度正規化 (Normalization)，在確保檢索精準度的前提下，大幅提升系統反應速度
        - 查詢擴張技術：應用 Gemini 進行語意詞擴張，補足使用者查詢詞（Query）的不足，提升搜尋的覆蓋率
    - 嚴謹的品質評估體系 (System Validation - 關鍵底氣)
        - NDCG 指標導入：為了解決系統推薦的合理性問題，你主導引入了 NDCG (Normalized Discounted Cumulative Gain) 指標，量化評估檢索結果的相關性排序
    - 專家驗證與去偏誤：
        - 建立「專家評分機制」，透過 3 位人類專家對影片相關性進行排序並取平均值，減少個人偏見
        - 迭代測試：透過比較 IDCG（理想得分）與系統得分，反覆更換 Embedding 模型進行測試，最終將系統得分率穩定提升至 88% 以上
- 系統技術棧
    - LLM & AI: Gemini Pro (語意擴張、內容摘要、測驗生成)
    - Fronted: Vue
    - Backend & Search: Python, FastAPI, MySQL, ChromaDB (向量資料庫)
    - Evaluations: NDCG 指標分析、多模型 Embedding 測試

## 查詢影片運作流程

- 下載-理解字幕-語意理解&向量化
1. 預先下載影片的字幕檔與標題，先藉由 gemini 模型進行語意理解，然後分別把向量化之後存入chomaDb
2. 使用者輸入 QUERY，利用 gemini 的語意詞擴張技術將其擴張
3. 利用開源語言模型將 QUERY 與擴張詞向量化
4. 計算歐幾里得距離，比對相似度，效果會接近餘弦相似度，但速度更快，然後進行排序推送給使用者
    1. 嘗試過計算餘弦相似度，但實際效果與速度都比較慢

## 學習地圖運作流程

- 使用者輸入自然語言，然後 gemini 生成合適的學習流程，透過流程內容尋找合適的影片
- 尋找合適影片的流程就跟搜尋影片一樣的流程

## 效果檢查：

> **NDCG**（Normalized Discounted Cumulative Gain，歸一化折損累計增益）是**資訊檢索、搜尋引擎與推薦系統中最常用來評估排序（Ranking）質量的指標**
> 
- 我提出透過 NDCG 指標來檢查查詢準確率
- 先在資料庫中只放入10部合格的影片，也就是系統可從這10部影片中選5部，但不可選中其他數百部影片
    - 若選到不相關影片之可能原因
        - 檢查資料問題(因為都是抓流量高的影片，代表有品質保證，不太可能是這個原因)
            - 標題或標籤誤導
            - 字幕為語音轉換而來，而轉換有誤
        - Embedding 模型問題(猜測是這個，所以換一個模型)
        - 搜尋系統設定問題
            - 索引演算法
            - 正規化疏失
                - 要用歐幾里得距離取代餘弦的話，要先將向量長度都正規化為1
- 接著透過人類專家看過每部影片，進行這10部影片與query的相似度排序，但為了避免個人偏見，因此將3位人類專家的排序進行平均
    1. 決定分數卡、計算滿分(IDCG)：將各專家之排序第一名給13分，其餘從9分往下遞減進行平均，不相關影片則以-100計算
    2. 計算系統得分
    3. 計算得分率，反覆更換模型進行測試，最終結果為至少88%


三層向量 RAG 架構： 針對 YouTube 資訊碎片化難以精確定位的痛點，開發標題/標籤/字幕分層嵌入系統，整合 Gemini Re-rank 與查詢擴張技術實現精準時間點定位，引入 NDCG 評估準確度達 88%
非同步 Self-Expanding 機制： 為解決知識庫初始內容不足的冷啟動問題，利用 asyncio 實作低信心值查詢自動觸發流程，背景執行影片下載與語意摘要，實現 ChromaDB 資料自動增量擴充
動態學習路徑與閉環： 改變傳統零散的學習方式，利用 LLM 生成階層式學習地圖並持久化至 PostgreSQL；整合 AI 自動化測驗與進度儀表板，完成從檢索到評量的完整學習工作流
全端系統開發與部署： 以 FastAPI 構建 30+ RESTful API，涵蓋 JWT 驗證與個人化推薦演算法；透過 Docker 容器化部署於 Railway 平台，確保 AI 應用具備商用級穩定性
